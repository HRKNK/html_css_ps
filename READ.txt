HTML - стандартизированный (общепринятый) язык гипертекстовой разметки.

- <!DOCTYPE html> - документ HTML5 стандарта
- <html> - ограничен <head> и <body> по спецификации

- <h1> - только один заголовок первого уровня на всю страницу, необходим для корректного чтения роботами/скринридером 
		далее: h2, h3
- <p> - параграф - текст/абзац
- <div> - тег группировки элементов (разделитель контента) / блочный элемент
- <span> - позволяет стилизовать часть текста / строчный элемент
- <sup> - надстрочный текст | - <sub> - подстрочный текст (чаще используется в формулах)

- <ul> - Unordered List - неупорядоченный список
- <ol> - Order List - упорядоченный список, который может содержать внутри себя теги <li>

Семантическая разметка – создание осмысленной и логически связанной информационной структуры содержимого (контента)
	- конкретное описание

- <header>, <footer>, <nav> - навигация, <section>, <article>, <aside> - боковая панель, 
		<main> - основная часть (содержит в себе секции, статьи и т.п.), <figure><figcaption> - коллекция изображений + пояснение

- Селекторный сниппет VSCode (составной селектор): header>nav>ul>li*5 | div.item{$}*12 - под заполнение

Формы:
- <form> - семантически-группируемый элемент (форма отправки)
- <label for=''> -  представляет собой подпись к элементу. Атрибут for определяет к какому тегу его отнести (или вложить элемент внутрь label)
		label фокусируется вместе со связанным элементом

- Атрибут 'name' в элементах внутри формы задают q-вери параметры при отправке формы
- <input type='radio' name='1'> - схожий атрибут 'name' объединяет все radio
- Атрибут 'required' (в элементах ввода input) делает их обязательными с точки зрения валидации (HTML5)
		аналогично: minlength | maxlength - минимальная | максимальная длинна, pattern - RegExp (идёт в связке с title - альтернативным описанием)

- Каскад CSS - применение стилей каскадом - сверху-вниз (последовательность классов* в HTML не влияет на каскад)
- Блочная модель (Box model) - это одна из основ в понимании CSS, модель форматирования документа. 
		Определяет формирование размеров элементов на странице, их взаимодействие между собой.
				Проще? - Блок состоящий из: контента, внутренний отступ, рамки, внешних отступов.

- Блочная модель занимает всю доступную ширину, а высота задается по контенту (если нет, то высота = 0);

	   * margin: 1 1 1 1 - по часовой стрелке начиная с 12:00
	   * margin: 1 1 - вертикально и горизонтально
	   * margin: 1 1 1 - верх, горизонт, низ

- Схлопывание внешних отступов - отступы margin-top и margin-bottom иногда объединяются в один, с размером равным наибольшему 
		из них (или размеру одного, если они равны).

- Поток документа: сверху-вниз и слева-направо (управление потоком: display, position)

CSS свойства:

- display - устанавливает поведение элементов в рамках потока:
		* none - скрывает элемент из потока,
		* block - поведение блока (div, p, h*),
		* inline - поведение строки (span, a, img, input),
		* inline-block - находится в строке, но ведет себя как блок,

- text-align: justify (межстрочное выравнивание текста/книжная верстка)

- vw - viewport width - (ширина относительно визуального окна)
- vh - viewport height - (высота относительно визуального окна)
- width - ширина
- height - высота

	* em - размер относительно родителя
	* rem - размер относительно корневого элемента html
	* width: 50% - ширина от родительского элемента
	* line-height: 120% - процент от размера шрифта

- margin - внешний отступ до соседнего элемента - exemple: margin: 0 auto (такая запись отцентрирует блок)
- padding - внутренний отступ от границы блока до контента

position: позиционирование элемента в потоке
	- static - применяется дэфолтно. top|left|bottom|right - недоступны
	- relative - двигается относительно своего изначального положения (static с доступом к свойствам - top|left|bottom|right)
		"оставляет за собой зарезервированное место"
	- absolute - относительно позиционирования родителя (родитель: relative) или окна браузера. top|left|bottom|right - доступны
		"НЕ оставляет/теряет за собой зарезервированное место"
	- fixed - только относительно окна браузера. Игнорирует позиционирование родителя (отличие с absolute)
	- sticky - (липкий) - комбинация fixed и relative.

	Для Гридов:
		- Ширина одного элемента изменяет ширину всего столбца (темплейта);
			.item { width 500px } / .wrapper { grid-template-columns: repeat(2, 1fr) } - получим для одного из столбцов 500px, вместо 1fr
		- Ширина указанная через "SPAN" может изменить ТЕМПЛЕЙТ при избыточном охвате
			grid-template-columns: repeat(2, 1fr) - состоит из 2-х колонок.
			айтем занимающий более 2х фрагментов заменит этот темплейт (grid-row: span 4 // до 4-х колонок) -> repeat(2, 1fr) auto auto
		- 'SPAN'(охват) работает только при указании шаблона grid-template-rows:
			.wrapper { grid-template-rows: repeat(6, 1fr) } / .item { grid-rows: span 3; }
		- auto - выстраивает автоматически по контенту
			Например: grid-template-columns: repeat(3, 1fr) auto; // 3 столбца по 1 фрагменту и 4-тый по его контенту

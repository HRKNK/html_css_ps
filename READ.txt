- <h1> - только один заголовок первого уровня на всю страницу, необходим для корректного чтения роботами/скринридером 
        далее: h2, h3

- <p> - параграф - текст/абзац
- <div> - тег группировки элементов (разделитель контента) / блочный элемент
- <span> - позволяет стилизовать часть текста / строчный элемент
- <sup> - надстрочный текст | - <sub> - подстрочный текст (чаще используется в формулах)

- <ul> - Unordered List - неупорядоченный список
- <ol> - Order List - упорядоченный список, который может содержать внутри себя теги <li>

Семантическая разметка – создание осмысленной и логически связанной информационной структуры содержимого (контента)
    - конкретное описание

- <header>, <footer>, <nav> - навигация, <section>, <article>, <aside> - боковая панель, <main> - основная часть (содержит в себе секции, статьи и т.п.)
- Селекторный сниппет VSCode: header>nav>ul>li*5

Формы:
- <form> - семантически-группируемый элемент (форма отправки)
- <label for=''> -  представляет собой подпись к элементу. Атрибут for определяет к какому тегу его отнести (или вложить элемент внутрь label)
        label фокусируется вместе со связанным элементом

- Атрибут 'name' в элементах внутри формы задают q-вери параметры при отправке формы
- <input type='radio' name='1'> - схожий атрибут 'name' объединяет все radio
- Атрибут 'required' (в элементах ввода input) делает их обязательными с точки зрения валидации (HTML5)
        аналогично: minlength | maxlength - минимальная | максимальная длинна, pattern - RegExp (идёт в связке с title - альтернативным описанием)


Стилизация текста:
- Абсолютные размеры: px, pt, cm, in, mm - не гарантируют свой размер (1px != 1 px), не всегда подходит к адаптиву
